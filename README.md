## 为什么一定要在买之前做搬瓦工测速？

搬瓦工目前在全球运营超过 20 个数据中心，覆盖美国、日本、香港、荷兰、新加坡、澳大利亚、加拿大、阿联酋等地。同样是"搬瓦工 VPS"，不同机房的延迟可以相差 10 倍：

所以，买之前测试，买了之后也要继续测试，这才是对的姿势。

## 场景二：已购买 VPS，想在服务器上测速

你已经有一台搬瓦工 VPS 了，想知道它的下载/上传速度和带宽表现，该怎么测？

### 方法 1：用搬瓦工演示站点做 SpeedTest

搬瓦工各机房都有对应的演示站点，直接访问就能在浏览器里测速：

| 机房 | 演示站点 |
|---|---|

| 日本大阪 | https://jpos.bwg.net |

测试页面会显示下载速度、上传速度、平均延迟、延迟波动等信息，一目了然。

### 方法 2：用 Speedtest-CLI 在 VPS 上测带宽

SSH 登录到你的 VPS，安装 Speedtest CLI 工具：

bash
curl -s https://packagecloud.io/install/repositories/ookla/speedtest-cli/script.deb.sh | sudo bash
sudo apt-get install speedtest
speedtest


这个命令会自动选择最近的测速节点，测出你的 VPS 对外带宽速度。想测到国内节点的速度，可以手动指定国内测速服务器 ID。

### 方法 3：用测速文件测下载速度

搬瓦工部分机房提供了可供下载的测速文件，直接用 wget 下载并计算速度：

bash
# DC2 机房 100MB 测试文件
wget http://23.252.96.201/100mb.bin

# DC4 机房 500MB 测试文件
wget http://98.142.136.11/500mb.bin

## 场景三：已购买，想知道自己走的是哪条线路

延迟测出来了，但更重要的是搞清楚你的流量到底走的是哪条路——CN2 GIA 还是普通 163？这决定了晚高峰的体验。

### 回程路由测试（backtrace 脚本）

在 VPS 上执行一键回程测试脚本：

bash
curl https://raw.githubusercontent.com/zhanghanyun/backtrace/main/install.sh -sSf | sh


执行后，脚本会列出你的 VPS 回程时经过的每一跳节点。看关键字：
- 出现 `59.43.x.x` → 走的是 CN2 GIA，恭喜你
- 出现 `202.97.x.x` → 走的是 CN2 GT，还行
- 出现 `163.x.x.x` 之类的 → 普通 163 骨干网，高峰期可能拥堵
